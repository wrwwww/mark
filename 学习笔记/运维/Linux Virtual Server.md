# **Linux Virtual Server** 是什么

**Linux Virtual Server (LVS)** 是一个由章文嵩博士于1998年发起的高性能、高可用性负载均衡项目，现已集成进 Linux 内核。

简单说，它相当于在四层（传输层）工作的超级流量分发器，能将海量网络请求转发给后端的真实服务器集群。

### 核心技术：三种工作模式

这是理解 LVS 的基石，它们决定了流量如何转发。

-   **NAT 模式 (地址转换)**
    **原理**：调度器在中间转换，既处理入站请求，也处理出站响应。
    **特点**：所有响应流量也必须经过调度器，因此它的带宽容易成为瓶颈，一般只能带 10-20 台服务器。

-   **DR 模式 (直接路由，最常用)**
    **原理**：请求经过调度器分发，后端服务器处理完后，直接将响应包发送给客户端，无需再绕回调度器。
    **核心技巧**：调度器和所有后端服务器必须处在同一个物理网段，并共享一个相同的虚拟IP（VIP），通过修改MAC地址来转发。
    **特点**：性能极强，是使用最广泛的模式。

-   **TUN 模式 (IP隧道)**
    **原理**：在请求包外再封装一层IP头。它突破了DR模式常见的二层网络限制，可以对不同地域的服务器进行负载均衡。

### 核心组件：ipvs 与 ipvsadm

-   **ipvs (IP Virtual Server)**：工作在 Linux 内核里的核心模块，真正干活的流量分发引擎。
-   **ipvsadm**：用户空间的命令行管理工具，用来给 ipvs 下发规则。

### 十种调度算法

LVS 的强大也在于灵活的调度策略，主要分两类：

**静态算法（不管服务器当前忙不忙，只按预设规则轮询）**

| 算法 | 原理 |
| :--- | :--- |
| **rr** (轮询) | 挨个分发，简单公平。 |
| **wrr** (加权轮询) | 性能强的服务器多分点活。 |
| **sh** (源地址哈希) | 来自同一个IP的请求始终发给同一台服务器，用于缓存或会话保持。 |
| **dh** (目标地址哈希) | 发往同一个目标IP的请求始终发给同一台服务器，常用于代理或网关。 |

**动态算法（根据服务器当前的连接数、负载来分配）**

| 算法 | 原理 |
| :--- | :--- |
| **lc** (最少连接) | 谁当前连接数少，请求就发给谁。 |
| **wlc** (加权最少连接) | 带权重的“最少连接”算法，是**默认算法**。 |
| **lblc** (基于局部性的最少连接) | 优先发给最近用过的服务器，兼顾负载均衡与缓存命中率。 |
| **dh** + 最少连接 | lblc 的复杂版，同时兼顾目标IP和连接数。 |
| **sed** (最短期望延迟) | 寻找处理请求最快的服务器，不太考虑权重。 |
| **nq** (永不排队) | 保证每个请求都能立刻找到一台服务器处理，从不排队。 |

### 高可用组合：Keepalived + LVS

LVS 本身存在单点故障：调度器一挂，整个集群就不可用了。实际生产环境中，常通过 **Keepalived** 来解决：

1.  **故障转移**：部署主、备两台调度器，通过 VRRP 协议虚拟出一个VIP。主调度器宕机时，备机自动接管，用户无感知。
2.  **健康检查**：Keepalived 能自动检测后端服务器是否健康，如果一台服务器宕机，就自动把它从转发列表里踢出去，恢复后再加回来。

### 动手实践的关键配置（DR模式为例）

这里给你一个最关键的、也最容易出错的配置要点：

**在后端服务器上，必须抑制ARP响应。** 因为VIP是共享的，如果不做抑制，后端服务器会直接响应客户端的ARP请求，导致流量绕过调度器直接到达服务器。

这是 Linux 下的典型抑制命令：
```bash
# 在后端服务器上执行
echo "1" > /proc/sys/net/ipv4/conf/lo/arp_ignore
echo "2" > /proc/sys/net/ipv4/conf/lo/arp_announce
echo "1" > /proc/sys/net/ipv4/conf/all/arp_ignore
echo "2" > /proc/sys/net/ipv4/conf/all/arp_announce
```
