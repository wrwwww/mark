## 一、最终结论（重点）

你使用的是：

> **AlmaLinux 10.0（EL10）**

在 **EL10（RHEL 10 / AlmaLinux 10 / Rocky 10）** 中：

### ✅ 这是**设计行为，不是异常**

|项目|EL9|**EL10（你现在）**|
|---|---|---|
|Java 模块（dnf module java）|✅ 有|❌ 已移除|
|openjdk-17|✅ 有|❌ 官方仓库不提供|
|openjdk-21|✅ 有|❌ 官方仓库不提供|
|openjdk-25|❌|✅ **唯一提供**|
|Java 策略|LTS 并存|**只保留主线**|

所以你遇到的所有现象都是 **EL10 的既定策略**：

- `dnf module reset java` ❌
    
- `dnf module list java` ❌
    
- `dnf install java-17-openjdk` ❌
    
- **只能看到 openjdk-25** ✅
    

---

## 二、这对 SeaTunnel 意味着什么（非常关键）

**Apache SeaTunnel（以及 Flink / Spark）目前的实际情况：**

- 官方主流支持：**Java 8 / 11 / 17**
    
- Java 21：部分可用
    
- **Java 25：未官方支持，存在真实风险**
    

👉 **在 EL10 上，绝对不建议使用系统自带的 Java 25 来跑 SeaTunnel**

---

## 三、EL10 上的「正确做法」（标准答案）

### ✅ 唯一推荐方案：**独立安装 LTS Java 17**

这是 **大数据生态在 EL10 上的统一实践**。

---

### 1️⃣ 下载 Temurin OpenJDK 17（Adoptium）

```bash
cd /opt
sudo mkdir -p jdk
cd jdk
```

下载（示例，版本号可更新）：

```bash
sudo curl -L -o jdk17.tar.gz \
https://github.com/adoptium/temurin17-binaries/releases/download/jdk-17.0.10+7/OpenJDK17U-jdk_x64_linux_hotspot_17.0.10_7.tar.gz
```

---

### 2️⃣ 解压并规范路径

```bash
sudo tar -xf jdk17.tar.gz
sudo ln -s jdk-17.* jdk-17
```

最终结构：

```text
/opt/jdk/
 ├── jdk-17 -> jdk-17.0.x+*
```

---

### 3️⃣ 为 SeaTunnel 单独指定 Java

**不要污染系统环境变量**，只针对 SeaTunnel：

编辑启动脚本（示例）：

```bash
vim /opt/apache-seatunnel/bin/seatunnel.sh
```

在顶部加入：

```bash
export JAVA_HOME=/opt/jdk/jdk-17
export PATH=$JAVA_HOME/bin:$PATH
```

---

### 4️⃣ 验证

```bash
/opt/jdk/jdk-17/bin/java -version
```

应输出：

```text
openjdk version "17.x.x"
```

再启动 SeaTunnel：

```bash
bin/seatunnel.sh
```

## 四、为什么 **不建议 alternatives / 系统级 Java**

在 EL10 上：

- 系统组件可能**依赖 Java 25**
    
- 替换系统 Java 存在**未来升级风险**
    
- Seatunnel 并不需要系统 Java
    

👉 **“应用自带 JDK” 是大数据领域的事实标准**
