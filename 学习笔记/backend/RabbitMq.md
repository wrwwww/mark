## 简介

**RabbitMQ** 是一个开源的消息代理和队列服务器，基于 **AMQP（高级消息队列协议）** 实现。它使用 **Erlang** 语言编写，具有高并发、分布式、可扩展的特性。

## 二、核心概念

1. **Producer（生产者）**：发送消息的程序
    
2. **Consumer（消费者）**：接收消息的程序
    
3. **Message（消息）**：由有效载荷（payload）和标签（label）组成
    
4. **Queue（队列）**：存储消息的缓冲区，遵循 FIFO 原则
    
5. **Exchange（交换机）**：接收生产者发送的消息，并根据规则路由到队列
    
6. **Binding（绑定）**：连接交换机和队列的规则
    
7. **Connection（连接）**：TCP 连接
    
8. **Channel（通道）**：虚拟连接，在连接中创建，减少 TCP 开销
    
9. **Virtual Host（虚拟主机）**：独立的消息环境，隔离权限

## Exchange 类型

| 类型          | 描述    | 路由规则                     |
| ----------- | ----- | ------------------------ |
| **Direct**  | 直连交换机 | routing key 完全匹配         |
| **Fanout**  | 广播交换机 | 忽略 routing key，广播到所有绑定队列 |
| **Topic**   | 主题交换机 | routing key 模式匹配（支持通配符）  |
| **Headers** | 头交换机  | 根据消息 headers 属性匹配        |
