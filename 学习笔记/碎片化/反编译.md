反编译 `war` 包中的 Java 类文件，通常是为了查看其中的源代码或进行调试。`war` 文件是一个 Web 应用程序的压缩文件，通常包含了 `.class` 文件（编译后的 Java 文件）。以下是如何反编译 `war` 包中的 Java 类文件的一些步骤：

### 1. 解压 `war` 文件

首先，你需要解压 `war` 包，以便获取其中的文件。可以使用命令行工具或者图形化工具来解压。例如：

```
# 使用命令行解压 war 文件
jar -xvf yourfile.war
```

或者你也可以使用图形化工具，如 WinRAR 或 7-Zip。

### 2. 查找 `.class` 文件

解压后，你会找到一系列的 `.class` 文件，这些文件是 Java 编译后的字节码。

### 3. 使用反编译工具

要将 `.class` 文件转回 Java 源代码，你可以使用以下反编译工具：

- **JD-GUI**：一个常用的 Java 反编译工具，可以将 `.class` 文件反编译成可读的 Java 源代码。

- 下载安装 JD-GUI。
- 打开 JD-GUI，加载你的 `.class` 文件或解压后的 `.jar` 文件（`.war` 文件本质上是 `.jar` 文件）。
- JD-GUI 会显示源代码，允许你查看或保存。

- **CFR**：另一个强大的 Java 反编译器，可以通过命令行运行。

- 下载 [CFR](http://www.benf.org/other/cfr/).
- 使用以下命令进行反编译：

```
java -jar cfr.jar yourfile.class
```

- **Procyon**：一个支持现代 Java 版本的反编译器，适合处理 Java 8 及更高版本的字节码。

- 下载 [Procyon](https://github.com/mstrobel/procyon).
- 使用以下命令反编译：

```
java -jar procyon-decompiler-0.5.30.jar yourfile.class
```

### 4. 检查反编译结果

反编译后的代码通常并不完美，可能会有一些缺失或不太易读的地方，特别是如果类文件经过了混淆处理。你可以尝试其他反编译工具或手动调整代码。

需要注意的是，反编译 Java 代码可能违反某些软件的许可协议，请在合法范围内使用这些工具。

如果你有更具体的需求或遇到问题，欢迎继续询问！

# Frida

是的，**Frida 完全支持无线连接**，只需通过 ADB 的无线调试功能实现。以下是具体步骤：

---

### **步骤 1：通过 USB 初始化无线 ADB 连接**

1. **连接设备到电脑**  
    用 USB 线将 Android 设备连接到电脑，确保已开启 **USB 调试**（在开发者选项中）。
2. **启用 ADB 的 TCP/IP 模式**  
    在终端运行以下命令，让设备监听某个端口（默认 `5555`）：

```
adb tcpip 5555
```

3. **获取设备的 IP 地址**

- 在设备上查看：**设置 → 关于手机 → 状态 → IP 地址**。
- 或通过 ADB 命令获取：

```
adb shell ip route
```

4. **通过 Wi-Fi 连接 ADB**  
    断开 USB 线后，运行以下命令（替换为设备的实际 IP）：

```
adb connect 192.168.1.100:5555  # 示例 IP
```

5. **验证 ADB 连接**

```
adb devices
```

如果看到设备的 IP 地址，说明无线 ADB 连接成功。

---

### **步骤 2：使用 Frida 无线调试**

1. **安装 Frida**  
    在电脑上安装 Frida 客户端：

```
pip install frida-tools
```

2. **在设备上安装 Frida Server**

- 下载对应架构的 [Frida Server](https://github.com/frida/frida/releases)（如 `frida-server-16.1.11-android-arm64.xz`）。
- 解压后推送到设备并赋予执行权限：

```
adb push frida-server /data/local/tmp/
adb shell "chmod 755 /data/local/tmp/frida-server"
```

3. **启动 Frida Server（无线模式）**  
    通过 ADB 无线连接启动服务：

```
adb shell "/data/local/tmp/frida-server &"
```

4. **通过 Frida 无线调试**  
    使用 `-H` 参数指定设备的无线 IP 和端口：

```
frida -H 192.168.1.100:5555 -f com.example.app --no-pause
```

- `-H`：指定无线连接的设备。
- `-f`：启动目标应用。
- `--no-pause`：立即启动应用。

---

### **常见问题与解决**

#### **1. 连接失败**

- **检查网络**：确保设备和电脑在同一局域网。
- **重启 ADB**：

```
adb kill-server && adb start-server
```

- **重新绑定端口**：

```
adb tcpip 5555  # 再次执行
adb connect 192.168.1.100:5555
```

#### **2. Frida 无法识别设备**

- **确认 Frida Server 版本**：客户端和服务端版本需一致。
- **检查 Frida Server 是否运行**：

```
adb shell "ps | grep frida-server"
```

#### **3. 防火墙或路由器限制**

- **开放端口**：确保设备的 `5555` 端口未被防火墙阻止。
- **静态 IP 分配**：在路由器中为设备分配固定 IP，避免 IP 变化导致断连。

---

### **注意事项**

- **安全性**：无线调试数据可能被同一网络内的其他设备截获，建议在可信网络中使用。
- **稳定性**：Wi-Fi 信号弱可能导致连接中断，建议靠近路由器使用。

---

### **总结**

通过无线 ADB 连接后，Frida 的使用方式与 USB 完全一致。只需确保设备与电脑在同一网络，并正确配置 IP 和端口即可享受无线调试的便利。
